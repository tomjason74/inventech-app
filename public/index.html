Digital Handoff & Security Accountability Policy

As a student of CCT College, you are personally responsible for the custody and safe handling of all IT hardware and software licenses assigned to you. By signing the digital handover signature pad upon checkout, you agree to the following conditions:

  • Equipment Security: You must not leave physical equipment unattended in public SCS labs or classrooms.
  • Software Licensing: Assigned license keys are for academic work on verified CCT assets only. Shared use or relocation of keys is strictly prohibited.
  • Decommission & Sanitization: Upon course graduation or status change, all assigned hardware must be checked back in. Assets will undergo standard NIST 800-88 data sanitization to scrub personal data.
  • Cryptographic Verification: Checkouts generate an immutable SHA256 receipt hash, establishing an audit trail in the PostgreSQL relational system.

Digital Accountability Policy

To comply with standard IT auditing rules, paper receipts have been entirely deprecated.

Every equipment handoff requires an interactive **E-Signature** which is merged with transaction timestamp and security credentials to generate a unique SHA256 cryptographic receipt hash.

Security Audit Trace:
- Unique barcoded Asset Tag
- Federated Active Directory ID
- Cryptographic signature data
- Hash stored permanently in PostgreSQL Core

Audit trail records are tamper-proof once signature receipt is saved.
